## 开发规范

### Git Commit 规范
遵循 Conventional Commits:

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具配置
```

### 分支策略
- `main`: 主分支，保持稳定
- `develop`: 开发分支
- `feature/*`: 功能分支

### 代码审查
- 每个阶段完成后进行代码审查
- 确保类型安全和代码质量
